# Wireframe и визуальный стиль MVP CRM для мастеров косоплетения

## 1. Главная идея интерфейса

CRM должна выглядеть не как сложная ERP-система, а как простое мобильное приложение для мастера.

Главный принцип:

```text
1 экран = 1 понятное действие
```

Основной путь пользователя:

```text
клиент → услуга → сложность → время → материалы → оплата → прибыль
```

---

## 2. Что берем из референса ERP-стиля

Из показанного интерфейса берем:

- чистые белые карточки
- аккуратные блоки с цифрами
- понятные статусы
- мягкие рамки
- светлый фон рабочей области
- крупные заголовки
- визуальную иерархию: главное крупно, детали ниже
- короткие сигналы “требует внимания”

Не берем:

- темный левый сайдбар
- перегруженное меню
- слишком много корпоративной терминологии
- плотные таблицы
- ощущение складской/бухгалтерской системы

---

## 3. Новый стиль MVP

Название стиля:

```text
Soft Beauty CRM
```

Характер:

- светлый
- чистый
- мягкий
- понятный
- не “бухгалтерский”
- с ощущением помощника для мастера

Цветовая логика:

```text
Фон приложения: #F7F8FA
Карточки: #FFFFFF
Основной текст: #111827
Вторичный текст: #6B7280
Акцент: сиреневый / розовый / теплый оранжевый
Успех: зеленый
Предупреждение: желтый / оранжевый
Ошибка / минус: мягкий красный
```

---

## 4. Навигация

Так как приложение mobile-first, основное меню должно быть снизу.

```text
┌─────────────────────────────┐
│                             │
│        Контент экрана       │
│                             │
├─────────────────────────────┤
│ Сегодня | Записи | Клиенты  │
│ Материалы | Финансы         │
└─────────────────────────────┘
```

Разделы:

1. Сегодня
2. Записи
3. Клиенты
4. Материалы
5. Финансы

│                             │
│ Материалы                   │
│ ┌─────────────────────────┐ │
│ │ Канекалон розовый       │ │
│ │ План: 2 | Факт: [3]     │ │

│ Оплата                      │
│ Цена услуги: [ 3 500 ₽ ]    │
│ Предоплата: [ 500 ₽ ]       │
│ Оплачено сейчас: [3000 ₽]   │
│                             │
│ Расчет                      │
│ Материалы: 770 ₽            │

# 14. Что должно быть в MVP

## Обязательно

- PWA
- IndexedDB
- демо-доступ 3 дня
- клиенты
- услуги
- записи
- сложность
- план/факт времени
- план/факт материалов
- приход материалов
- автоматическое списание материалов
- финансы: сегодня / месяц / период

- импорт базы

# 16. Главный ориентир

Приложение должно отвечать мастеру на 3 вопроса:

```text
Кто ко мне записан?
Сколько материала я потратила?
Сколько я реально заработала?
```
